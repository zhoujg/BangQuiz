# 浏览器缓存清理说明

## 问题描述
如果看到类似以下错误：
```
ERROR RuntimeError: NG04002: Cannot match any routes. URL Segment: 'mock-exam'
```

这通常是因为浏览器缓存了旧版本的代码。

## 解决方法

### 方法1: 硬刷新（推荐）
**Windows/Linux:**
- Chrome/Edge: `Ctrl + Shift + R` 或 `Ctrl + F5`
- Firefox: `Ctrl + Shift + R`

**macOS:**
- Chrome/Edge: `Cmd + Shift + R`
- Firefox: `Cmd + Shift + R`
- Safari: `Cmd + Option + R`

### 方法2: 清除浏览器缓存
**Chrome/Edge:**
1. 按 `F12` 打开开发者工具
2. 右键点击刷新按钮
3. 选择"清空缓存并硬性重新加载"

**Firefox:**
1. 按 `Ctrl + Shift + Delete` (Windows) 或 `Cmd + Shift + Delete` (Mac)
2. 选择"缓存"
3. 点击"立即清除"

**Safari:**
1. 按 `Cmd + Option + E` 清空缓存
2. 刷新页面

### 方法3: 使用无痕/隐私模式
在无痕模式下打开应用，这样不会使用缓存：
- Chrome/Edge: `Ctrl + Shift + N` (Windows) 或 `Cmd + Shift + N` (Mac)
- Firefox: `Ctrl + Shift + P` (Windows) 或 `Cmd + Shift + P` (Mac)
- Safari: `Cmd + Shift + N`

### 方法4: 开发者工具禁用缓存
1. 按 `F12` 打开开发者工具
2. 打开 Network (网络) 标签
3. 勾选 "Disable cache" (禁用缓存)
4. 保持开发者工具打开状态
5. 刷新页面

## 验证修复
清除缓存后，应该能够：
1. ✅ 点击"模拟考试"按钮弹出选择对话框
2. ✅ 点击"ADM地图"显示"功能开发中"提示

4. ✅ 所有页面都有返回按钮

## 开发环境注意事项
如果你是开发者，建议：
1. 始终在开发者工具中启用"Disable cache"
2. 使用 Angular 的热重载功能（应该自动生效）
3. 如果热重载不工作，重启前端开发服务器：
   ```bash
   cd frontend
   # 停止当前服务器 (Ctrl+C)
   npm start
   ```

## 常见问题

### Q: 为什么会有缓存问题？
A: Angular 应用是单页应用(SPA)，浏览器会缓存 JavaScript 文件以提高性能。当代码更新时，浏览器可能仍在使用旧的缓存文件。

### Q: 每次更新都需要清除缓存吗？
A: 在开发环境中，Angular 的热重载通常会自动更新。但有时需要手动刷新。在生产环境中，应该配置适当的缓存策略。

### Q: 如何避免用户遇到缓存问题？
A: 在生产环境中，可以：
1. 使用版本化的文件名（Angular 默认会做）
2. 配置适当的 HTTP 缓存头
3. 使用 Service Worker 进行缓存管理
